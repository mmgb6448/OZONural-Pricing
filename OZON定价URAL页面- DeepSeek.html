            const profitMin = getInputAsNumber(profitMinInput, 10) / 100;
            const profitPre = getInputAsNumber(profitPreInput, 50) / 100;

            // 基础计算
            const totalCost = purchasePrice * quantity;      // 采购总成本
            const weightKg = weightGram / 1000;              // 转为kg

            // 存储符合条件渠道及对应的人民币售价 (用于比较)

                // 3. 计算该渠道下的售价 (使用售价利润率)

                if (salePriceRub < ch.priceMinRub - 1e-9 || salePriceRub > ch.priceMaxRub + 1e-9) continue;

                availableChannels.push({

                    logisticFee: logisticFee,
                    saleCny: salePriceCny,
                    saleRub: salePriceRub

            const denominatorMin = 1 - (loss + commission + ad + profitMin);

            const minCny = numerator / denominatorMin;
            const saleCny = numerator / denominatorSale;
            const preCny = numerator / denominatorPre;

            const minRub = minCny * rate;
            const saleRub = saleCny * rate;
            const preRub = preCny * rate;

输入框太长了，不太美观

我们收到了用户的新反馈：“输入框太长了，不太美观”。用户希望输入框的长度进行调整，使其更美观。之前的版本中，输入框确实比较宽，因为使用了grid布局，每个输入框占据了较宽的列。现在需要调整输入框的宽度，使其看起来更紧凑，同时保持核心输入框的放大效果（采购单价、采购数量、克重）但整体宽度要缩小。

用户的要求是“输入框太长了，不太美观”，所以我们需要调整输入框的宽度。我们可以考虑：

  • 限制每个输入框的最大宽度，例如设置max-width: 200px 或使用更紧凑的列宽。

  • 调整grid的列宽，比如使用固定宽度或者使用auto-fit但加上最大宽度。

  • 确保三个核心输入框仍然突出，但宽度不要过长。

可能还需要调整布局，使整体更平衡。可以设置每个输入框的宽度为固定像素，或者使用百分比但限制最大宽度。也可以在父容器上加一个最大宽度，让输入框不那么拉伸。

同时，保持分组的美观。我们可以在CSS中对.input-item input设置一个max-width，或者对.param-grid设置grid-template-columns为repeat(auto-fit, minmax(150px, 1fr))，并加上max-width限制。

另外，用户可能希望核心输入框仍然比其他的稍微大一点，但整体长度适中。我们可以保留.core-field input的字体和内边距，但控制宽度，比如宽度为100%但父元素有最大宽度。

  • 在.card上设置max-width适当，比如1200px。

  • 对于.core-field input，可以设置max-width: 260px 稍大一些。

  • 调整grid的间距，使整体紧凑。

我们还需要注意，不要让输入框变得太窄导致内容显示不全。保留足够的宽度。
